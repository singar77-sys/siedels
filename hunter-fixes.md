# ENTRY 4 — PREMIUM (Psychology of Premium Websites)

Halo effect / cognitive load / peak-end rule. Code + site combined.

## HERO / STAY SHARP — HALO EFFECT (50ms test)

- [ ] Hero fails halo — Ken Burns zoom on a brightness(0.32) mush is the only motion; no visual anchor commands the eye
- [ ] `filter: brightness(0.32)` in HeroPanel.tsx crushes the photo into black soup — the barber pole stops being a religious object
- [ ] Both hero images ship to DOM (HERO_LIGHT + HERO_DARK, one display:none per theme) — CSS theme-swap is clever code, but doubles LCP candidate weight
- [ ] Only `HERO_LIGHT` has `priority` — dark-theme users get a lazy hero. Pick one priority image per theme via next/image conditional, not CSS hide
- [ ] Two centers of gravity in first viewport — STAY SHARP headline AND the bottom-right testimonial card compete. Halo needs ONE feeling, not a split vote
- [ ] No Spline/3D/Nano Banana scroll sequence — Apple-level halo requires a crafted visual; Ken Burns is 2011 craft
- [ ] Font pairing Khand + Hind is fine but generic; neither carries the halo weight a $96 haircut shop earns

## HERO COPY & CTAs — HALO

- [ ] "IN MEMORY OF CARL SIEDEL" eyebrow is buried at stagger-1 at 11px/0.3em — the halo emotional anchor is literally the smallest text on screen
- [ ] STAY + SHARP split-color trick (solid + stroke) is the one good halo beat — keep
- [ ] Dual CTA (primary tan + ghost border) dilutes the first action — halo sites resolve to ONE dominant verb
- [ ] "SWIPE" chevron-pulse at bottom center competes with the real CTA. Either commit to it as primary wayfinding or kill it

## COGNITIVE LOAD — NAV / HEADER

- [ ] Header packs 9 elements (logo + spinning-pole SVG + 6 nav dots + progress line + BOOK NOW + theme toggle) into 96-128px of height — cognitive load exceeds premium threshold
- [ ] The 6-dot progress indicator + 6 text labels + animated track fill is triple-redundant — one signal says where you are, not three
- [ ] `PANELS = ['HOME','TEAM','WORK','SERVICES','SCHEDULE','CONTACT']` — six equal-weight panels fight for attention. Premium = one primary path
- [ ] Pole SVG has 12 animated lines + clipPath + cap rects — lovely craft, but it's in the header NOT the hero. Move it where halo can use it

## COGNITIVE LOAD — ARCHITECTURE

- [ ] Horizontal snap-x carousel with 6 panels is novel but violates cognitive fluency — users expect vertical scroll. Novelty tax = load
- [ ] Arrow key listener binds window globally — fine, but ArrowDown/Up scrolling per-panel is a hidden affordance no one will find
- [ ] IntersectionObserver threshold 0.6 + scrollTo behavior:smooth is good, but mobile users fight the snap to reach content
- [ ] Fixed left/right red-triangle "previous/next" buttons render only md: — mobile users get a `swipe_right` Material icon; desktop users get CSS triangles. Two different UX languages on one site

## COGNITIVE LOAD — PANELS

- [ ] Panel height locked to viewport (`h-dvh`) means every section must fit in 100dvh or scroll internally — forces content compression = visual density
- [ ] GalleryPanel + TeamPanel + ServicesPanel all have their own internal scroll — three different overflow behaviors in one site
- [ ] ContactPanel + footer both carry phone/address — duplication increases load, reduces clarity

## PEAK-END RULE — MICRO-INTERACTIONS (missing)

- [ ] No Framer Motion. No GSAP. package.json: just next, react, react-dom. Zero animation libraries = zero orchestrated peaks
- [ ] Motion library is 5 CSS keyframes total: ken-burns, pole-spin, hero-rise, hero-slide, ticker, theme-reveal. Hover states use `transition-colors` only
- [ ] Every button = `hover:text-red` or `hover:bg-red-hover` — no scale, no lift, no shadow, no spring. Peak moments absent
- [ ] Team cards have no card-lift — user clicks into Modal with no anticipation beat
- [ ] Service cards same — zero hover choreography between grid and modal
- [ ] Gallery grid has no hover reveal, no zoom, no attribution-on-hover — just static images
- [ ] Modal open/close has no transition — it pops in. Premium modals breathe
- [ ] Image `onError` in service modal just sets `display: none` — a peak-end fail state with no fallback craft

## PEAK-END RULE — THE ONE WIN

- [ ] Theme toggle uses View Transitions API with clip-path circle reveal from toggle coordinates — this is the single most premium moment on the site. Keep and expand
- [ ] `::view-transition-new(root)` at 0.6s cubic-bezier is genuine craft — use the same vocabulary for panel transitions and modal opens

## PEAK-END RULE — PROPOSED ADDS

- [ ] Add Framer Motion for button hover scale (1.0 → 1.02, 200ms), card lift (-4px translateY + shadow)
- [ ] Add GSAP ScrollTrigger or IntersectionObserver reveals for in-panel content — right now every panel's content just sits there
- [ ] Add a Spline or Three.js barber pole in hero — the literal religious object becomes the halo anchor
- [ ] Add Nano Banana 2 + Kling scroll sequence for the Work/Gallery panel — "haircut arrives as you scroll" is the one premium apex for this business
- [ ] Add Lenis for smooth scroll on the vertical axis of internal panels

## TYPOGRAPHY / DESIGN TOKENS

- [ ] No spacing scale named — premium systems have `--space-1` through `--space-12` tokens, Tailwind defaults are used everywhere

## IMAGERY

- [ ] No video in hero — premium halo often uses a 6-8s silent loop (pole spinning, clippers, straight razor strop). Current Ken Burns is a still photo zoom

## SYSTEM WEAK POINTS (critical → minor)

### Critical
- [ ] No analytics/telemetry stack — can't answer "which barber converts best?" or "where do users drop off?"

# ENTRY 20 — GOD-MODE (Principal Engineer Code Audit)

## 1. EXECUTIVE VERDICT

**Competent. Not elite.** The Next.js 16 + React 19 foundation is modern and the JSON-LD / theme-system / Square per-barber wiring show taste. But under the hood the site is a **client-heavy single-route SPA dressed as a multi-panel scroll experience**. The App Router is barely being used as an App Router — one `/` route doing the work of twelve. Hydration cost, bundle bloat, SEO ceiling, and maintainability all pay the tax. **Grade: B− on craft, C+ on architecture, D on server-first.**

## 2. WHAT IS WEAK

### Architecture
- [ ] **Single-route monolith.** `/` hosts HOME / TEAM / WORK / SERVICES / SCHEDULE / CONTACT as scroll panels. App Router's entire superpower (nested layouts, per-route streaming, RSC boundaries, parallel routes) is unused.
- [ ] **11 Square widgets all mounted on one route.** Every barber's iframe config ships on first load whether the user ever reaches SCHEDULE or not.
- [ ] **No route segments for barbers, services, or locations.** `/barbers/[slug]`, `/services/[slug]`, `/locations/medina` do not exist. SEO surface area is ~1 page for an 11-person shop.
- [ ] **Client-component bleed.** Nearly every component is probably `"use client"` because of the IntersectionObserver / theme toggle / scroll logic. Should be RSC by default with tiny client islands.
- [ ] **Data lives in `src/data/shop.ts` as a TS constant.** Fine for v1. Becomes a ceiling the moment a barber changes bio, price, or photo and a non-dev needs to edit it.
- [ ] **No Suspense boundaries.** Square iframes block nothing and reveal nothing; they just pop in.
- [ ] **Theme pre-hydration inline script is doing the right thing but is probably untyped and duplicated across pages.**

### Performance
- [ ] **Material Symbols Outlined loaded via blocking `<link rel="stylesheet">`.** ~500KB of icon font. Render-blocking. Entry 18 already flagged — re-flagging here because it's a God-Mode-level offense.
- [ ] **Dual-mount HERO_LIGHT + HERO_DARK.** Both images ship on first paint. One is always invisible. 100% waste on one hero image.
- [ ] **Square iframes ×11 on a single route.** Every iframe boots its own micro-app. First SCHEDULE scroll = serial network storm.
- [ ] **`revalidate = 1800` on home.** Correct instinct, but unused because nothing on home is dynamic server data. Revalidation is decorative.
- [ ] **No image component discipline.** Hero images are likely not served via `next/image` with proper `sizes` + priority + AVIF.
- [ ] **Fonts loaded via `next/font` (good) but Khand / Hind are doing bodywork Inter Tight / Bebas Neue would do in half the payload** — see Entry 18.
- [ ] **Snap-x scroll carousel forces all 6 panels to be in the DOM simultaneously.** No panel lazy-mount. All code, all images, all assets, all the time.
- [ ] **IntersectionObserver at threshold 0.6 fires on every panel change and likely triggers React re-renders instead of direct DOM mutation.**

### UX / Interface
- [ ] **No URL reflects panel state.** `/#schedule` or `/schedule` would let users deep-link, share, bookmark. Currently the URL is always `/`.
- [ ] **Back button doesn't navigate between panels.** Users expect browser history to work. It doesn't.
- [ ] **No loading state on Square iframes** (flagged Entry 19).
- [ ] **No empty state for "no availability this week."** What does SCHEDULE show when Square says no slots?
- [ ] **Form errors on Formspree subscribe are probably toast-less.** Submit and hope.

### Accessibility
- [ ] **Snap-x horizontal scroll is a keyboard nightmare.** Tab order, focus trapping, skip-links — likely all broken.
- [ ] **Panel headings may not form a clean H1 → H2 → H3 hierarchy** across all 6 panels. SEO + screen-reader damage.
- [ ] **View Transitions clip-path on theme toggle is probably not gated on `prefers-reduced-motion`.**
- [ ] **Stroked SVG SHARP text** — `aria-label` status unknown. If unlabeled, it's a decorative void for screen readers.

### SEO / Crawlability
- [ ] **1 URL, 11 barbers, 4+ services, 2 locations.** Google indexes what Google can crawl. Entry 17 fixed this conceptually; God-Mode flags it as an **architecture** problem, not a content problem.
- [ ] **JSON-LD is good** (BarberShop, aggregateRating, hours, geo). Could be extended with `Person` schema per barber on new routes.
- [ ] **No sitemap.xml / robots.txt discipline visible.** App Router has `app/sitemap.ts` and `app/robots.ts` — use them.
- [ ] **No Open Graph image per panel / barber / service.** Every share looks identical.

### Security / Robustness

- [ ] **Square iframe IDs in `src/data/shop.ts`** — fine, they're public, but the pattern assumes they never rotate. Make it env-backed or CMS-backed for resilience.
- [ ] **No error boundaries visible** around Square iframes. If Square fails, the SCHEDULE panel dies silently.

### `app/api/subscribe/route.ts` — hardened form handler

```ts
import { NextResponse } from 'next/server'
import { Ratelimit } from '@upstash/ratelimit'
import { Redis } from '@upstash/redis'

const ratelimit = new Ratelimit({
  redis: Redis.fromEnv(),
  limiter: Ratelimit.slidingWindow(5, '10 m'),
})

export async function POST(req: Request) {
  const ip = req.headers.get('x-forwarded-for') ?? 'anon'
  const { success } = await ratelimit.limit(ip)
  if (!success) return NextResponse.json({ error: 'Too many' }, { status: 429 })

  const form = await req.formData()
  if (form.get('company')) return NextResponse.json({ ok: true }) // honeypot
  const email = String(form.get('email') ?? '')
  if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)) {
    return NextResponse.json({ error: 'Invalid email' }, { status: 400 })
  }

  const res = await fetch('https://formspree.io/f/meepnyaj', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json', Accept: 'application/json' },
    body: JSON.stringify({ email }),
  })
  if (!res.ok) return NextResponse.json({ error: 'Upstream' }, { status: 502 })
  return NextResponse.json({ ok: true })
}
```

## ENTRY 26 — /stop-slop (AI-Writing-Pattern Audit on Site Copy)

> Skill: Eliminate AI prose tells. Active voice. Specifics. No em dashes. No three-beat throat-clearing. No quotable bait.
> Frame: Audit every line of human-readable copy on the site. Score it. Flag the LLM tells. Rewrite the worst.

---

### THE COPY UNDER REVIEW

Hero strings, meta descriptions, FAQ answers, label text, button copy, fine print, JSON-LD descriptions, and barber bios across `layout.tsx`, all `page.tsx` files, and `data/shop.ts`.

---

### TELLS FOUND

**1. Em dashes everywhere.**
The skill rules forbid em dashes. The site uses them as decoration:
- `"Thanks — Siedel's Barbershop"` (page title pattern, /thanks)
- `"Cash only — ATM on site"` (frequent body usage)
- `"982 N COURT STREET · MEDINA, OHIO"` (middle dot, acceptable)
- `"Open Monday–Saturday"` (en dash, acceptable in date ranges)

The em dashes are the signature. Strip them. Use periods. Use line breaks. Trust the reader.

**2. Three-beat formula.**
`/thanks/page.tsx` line 22:
> "Tap your barber. Drop a quick Google review. It helps the whole crew."

Three short sentences, ascending intimacy, the third softer than the second. Pure LLM cadence. The "it helps the whole crew" is throat-clearing for "leave a review." Cut it.

**Fix:** "Pick your barber. Leave a review."

**3. Vague declarative.**
`/team/page.tsx` meta description:
> "11 barbers and stylists on staff"

What is a "stylist" here? Is it a barber with a different license? Different chair? The site never says. Naming the difference would specify. Hiding behind "and stylists" is the LLM hedge.

**Fix:** "11 barbers cutting in Medina, Ohio." (Drop "stylists" if it isn't a real distinction. Add it as its own role if it is.)

**4. Existential weak verb.**
`/services` FAQ answer:
> "There is an ATM on site."

Passive existential. The shop owns the ATM. Make the shop the actor.

**Fix:** "We have an ATM."

**5. Quotable bait.**
`/thanks/[slug]/page.tsx` line 51:
> "Your review helps the whole crew. If {firstName} made your day, mention them by name."

Two sentences. The first is a pull-quote. The second adds the only useful information. Cut the first.

**Fix:** "If {firstName} made your day, mention them by name."

**6. Vague freshness claim.**
`/schedule` body:
> "Updated weekly"

Updated when? By whom? The page revalidates every 30 minutes. The line is wrong on its face. Either give the timestamp or drop the claim.

**Fix:** "Last synced {fetchedAt}." (Already rendered below — promote it, delete the contradictory line.)

**7. Duplicate boilerplate across meta tags.**
`layout.tsx` ships the same string in `description`, `openGraph.description`, and `twitter.description`:
> "Eleven barbers in Medina, Ohio. Cash only. ATM on site."

Copy-paste-and-pray. Each surface has different audience and length budget. Tailor them.

**Fix:**
- `description` (Google snippet, 155 chars): "Eleven barbers in Medina, Ohio. Haircuts, fades, beard work. Cash only. Walk-ins welcome. 330-952-0777."
- `openGraph.description` (social card, room to breathe): "Eleven chairs on Court Street. Cash only. ATM on site. Open six days."
- `twitter.description`: same as OG.

**8. Pierre's bio math contradiction.**
`shop.ts`:
> "26 years behind the chair. Barber school graduate, class of 2003."

Today is April 2026. 2026 − 2003 = 23. Either the bio lies or the year lies. Pick one. (Already flagged in Conty Entry 22; included here because it's a copy issue too — readers do the math.)

**9. Label-style throat-clearing.**
`/thanks` and `/thanks/[slug]`:
> "THANKS FOR STOPPING IN"
> "THANKS FOR TRUSTING"

These read like greeting-card label fonts. Fine on a card. On a digital page they delay the actual content. Use them once or drop them.

**10. Adverbs.**
`/thanks/page.tsx`:
> "Drop a **quick** Google review."

"Quick" is selling the action's effort. Don't sell. Ask. Cut "quick."

### SILENT FAILURE DETECTOR — THE LIST
